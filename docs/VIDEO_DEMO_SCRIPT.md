# Video Demo Script

A short script and shot list for creators making a YouTube short, Instagram reel, or TikTok about AI You.

The promise of the video is simple: **ChatGPT forgets. GitHub remembers. Here's a free fix you can install in 10 minutes.**

The most relatable use case is **relationship continuity**. Everyone understands the pain of explaining the same situation to a chatbot for the tenth time.

## Recommended length

- 60–90 seconds for shorts / reels.
- 5–8 minutes for a long-form YouTube video that walks through setup.

## Hook (first 3 seconds)

Open with the friction. Pick one:

- "ChatGPT keeps forgetting your wife's name. Here's how to fix it."
- "Your AI loses everything between chats. This GitHub trick fixes it."
- "I taught ChatGPT to remember me. The whole thing is free."
- "Tired of explaining yourself to ChatGPT every time? Watch this."

Show your face for the hook. Talking-head outperforms screen-only here.

## Beat-by-beat structure (60–90s reel)

| Time | Beat | What's on screen |
| --- | --- | --- |
| 0:00 | Hook | Talking-head, frustrated. "ChatGPT forgot my partner's name. Again." |
| 0:05 | Stakes | Cut to a fake chat where ChatGPT asks "who is Sam?" after a long previous chat. |
| 0:12 | Reveal | "GitHub remembers what ChatGPT forgets." Show the public foundation repo (`utb2017/ai-you`). |
| 0:20 | Steps | Quick cuts: "1. GitHub account. 2. Connect GitHub to ChatGPT. 3. Click Use this template." |
| 0:30 | Setup prompt | Paste setup prompt into ChatGPT. Show the smoke-test file appearing in the private repo. |
| 0:45 | Archive demo | A short relationship-style chat. Say `Archive GitHub`. Cut to the new Markdown note appearing in `categories/relationship/`. |
| 0:58 | Recall demo | New chat. Say `Recall Context`. Cut to ChatGPT replying with "I pulled your AI You context. The relevant pattern is …" |
| 1:10 | Close | "It's free. The repo's in the description. Go remember yourself." |
| 1:15 | End card | URL: `github.com/utb2017/ai-you`. Two commands on screen. |

## Demo data: do not use your real life

When filming, **use a fake demo repo with invented names**. Reasons:

- Your real memory repo is private for a reason.
- You don't want a frame-grab from your video to show your real partner's name, your real symptoms, your real client list.
- Invented data also looks cleaner and more focused on screen.

Pick a fictional persona for the demo:

- "Sam" — a partner.
- "Mark" — a boss.
- "Iris" — a kid.
- "Dr. Patel" — a doctor.

Stay consistent across cuts.

## Long-form structure (5–8 min YouTube)

1. **Cold open (0:00–0:30):** the friction, the promise, what you'll show.
2. **Why this matters (0:30–1:30):** the real cost of starting over with ChatGPT every chat. Tie it to a viewer pain point.
3. **The two commands (1:30–2:30):** explain `Archive GitHub` and `Recall Context` in plain words.
4. **Setup walkthrough (2:30–5:00):** screen-record the entire flow from GitHub login to smoke test.
5. **Archive demo (5:00–6:00):** real-feeling chat with a fictional persona. Save it. Show the resulting Markdown.
6. **Recall demo (6:00–7:00):** open a new chat. Recall. Show how ChatGPT carries continuity.
7. **Privacy and trust (7:00–7:30):** "Your repo is private. We don't run a server. You can delete the whole thing whenever you want."
8. **Call to action (7:30–end):** "Link in description. Try it on one chat tonight. Tell me how it goes."

## Captions / on-screen text suggestions

- "Two commands. That's the whole product."
- "Free. No app. No account."
- "Your memory lives in your GitHub. Not ours."
- "ChatGPT can't remember. GitHub can."
- "If you can copy and paste, you can do this."

## Timing tips

- Record the setup once cleanly. It's the slowest part. A re-take saves you in the edit.
- Use jump cuts during the GitHub clicking; nobody wants to watch a "Use this template" page load in real time.
- Keep the archive demo short. The point is the **result** (a new Markdown file appearing), not the chat itself.

## Risks to call out on camera

If you want to be honest with viewers (recommended):

- "Some ChatGPT model surfaces hide the GitHub write tool. If yours does, switch model and rerun the setup prompt."
- "Your repo is private, but assume anything you put in it is recoverable. Redact what shouldn't be there."

That kind of honest framing earns trust faster than overselling.
